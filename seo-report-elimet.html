Технический SEO

40 / 100

Sitemap

  • Критично
    Все URL в sitemap используют HTTP вместо HTTPS
    sitemap-iblock-2.xml (53 статьи) и другие файлы содержат URL с протоколом http://. Google считает их отдельными от HTTPS-страниц и может индексировать устаревшие версии. Исправить через настройки SEO-модуля Битрикс → перегенерировать sitemap.
  • Высокий
    Нестандартное расположение sitemap: /custom-sitemap/
    Стандартный путь /sitemap.xml не возвращает файл. Большинство SEO-инструментов и краулеров проверяют именно его. Добавить редирект /sitemap.xml → /custom-sitemap/ или создать alias.
  • Критично
    Технические URL в sitemap-iblock-3.xml
    Файл содержит 3 записи вида /iblocks/detail.php?ID=5 — внутренние Битрикс-страницы без SEO-ценности. Эти URL должны быть закрыты от индексации и удалены из sitemap.

Robots.txt

  • ✓ Административные разделы Битрикс заблокированы
  • ✓ Clean-param настроен (utm_source, yclid, PAGEN_1…)
  • ✓ Ссылка на sitemap: Sitemap: https://elimet.ru/custom-sitemap/
  • ✓ Разрешены /llms.txt и /llms-full.txt

404-страницы в навигации

  • Критично
    /cooperation/ и /production/ → HTTP 404
    Оба раздела присутствуют в главном меню сайта, но возвращают 404. Это прямые потери трафика, снижение авторитета сайта в глазах краулеров и плохой пользовательский опыт. Восстановить страницы или настроить 301-редирект.

Canonical и мобильность

  • ✗ Canonical URL не обнаружен ни на одной странице — риск дублей
  • ✗ Viewport meta-тег не найден — возможны проблемы с мобильным ранжированием
