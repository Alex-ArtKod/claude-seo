# GEO/AEO Batch — пакет из 8 материалов (Дзен + Telegram + DOCX)

## Назначение

Один запуск = **N тем** (по умолчанию **8**). На каждую тему:
- лонгрид для **Яндекс.Дзен** (пайплайн `seo-writer`, фазы 0–4);
- короткий пост для **Telegram** (пайплайн `post-writer`);
- **DOCX** лонгрида (Word COM через `seo-writer/references/md-to-docx.ps1`).

Контент для **Artkod** (SEO / GEO / AEO), опирается на вирусные референсы X.com и
практические чеклисты, без хайпа и без гарантий «попадания в ChatGPT».

**Текущее состояние проекта:** 40 материалов (01–40). Следующий запуск — **41–48**.

## Оркестратор (основной поток)

`N` = аргумент (default 8). `PROJECT` = папка проекта (default выше).

### Шаг 0 — Подготовка структуры

Убедиться, что существуют папки: `ТЗ`, `Лонгриды Дзен`, `ТГ Посты`, `DOCX`,
`_исходники`. В корне — `project-context.md`, `rules.md`.

Определить `START_NN` = max(номер в `ТЗ/`) + 1.

### Шаг 1 — N новых тем

Сформулировать **N уникальных тем** по SEO / GEO / AEO:
- чеклист углов: `references/topic-angles.md`;
- опираться на референсы (Lily Ray, Gaetano, Aleyda, Ethan Smith, Cyrus Shepard,
  корпус `post-writer/examples/corpus.md`);
- не повторять уже опубликованные заголовки в `ТЗ/` и `Лонгриды Дзен/`;
- угол Artkod: B2B, честная методология, связка SEO+GEO+SERM;
- избегать перегретых «SEO агентство Москва», «что такое SEO»;
- в одной восьмёрке — **разные углы**, не 8 вариаций одной механики.

Для каждой темы записать `ТЗ/ТЗ-NN.md`:
- H1, объём лонгрида, ключевые запросы, референс, список H2 с ~знаками,
  ограничения (NEVER из rules.md, без гарантий ChatGPT), CTA Artkod.

Показать пользователю краткую таблицу тем (NN + заголовок).

### Шаг 2 — Рабочие папки субагентов

Для каждого NN в `START_NN … START_NN+N-1`:
```
_исходники/ТЗ-NN/
  project-context.md  (копия)
  rules.md            (копия)
  ТЗ.md               (копия из ТЗ/ТЗ-NN.md)
```

### Шаг 3 — Запуск субагентов

По **3 агента** за раз (`subagent_type: generalPurpose`, `model: composer-2.5-fast`).

Промпт субагента — шаблон из `references/subagent-prompt.md`, подставить NN и пути.

Каждый субагент обязан сохранить:
| Файл | Куда |
|------|------|
| `article-plan.md` | `_исходники/ТЗ-NN/` |
| `article-final.md` | `_исходники/ТЗ-NN/` |
| `telegram-post.md` | `_исходники/ТЗ-NN/` (2 хука + готовый пост) |
| копия лонгрида | `Лонгриды Дзен/NN — {короткий H1}.md` |
| копия ТГ | `ТГ Посты/NN — {короткий H1}.md` |

**Без подпапок** внутри `Лонгриды Дзен` и `ТГ Посты`.

### Шаг 4 — DOCX

Для каждого нового файла в `Лонгриды Дзен/NN — *.md`:

1. Скопировать `seo-writer/references/md-to-docx.ps1` → `C:\Temp\md_to_docx_run.ps1`
2. Пересохранить скрипт в **UTF-8 BOM**
3. Запустить Word COM:
   ```powershell
   powershell -ExecutionPolicy Bypass -File C:\Temp\md_to_docx_run.ps1 `
     -InputFile "{путь к md}" `
     -OutputFile "{PROJECT}\DOCX\{то же имя}.docx"
   ```
4. При блокировке Word: `Stop-Process WINWORD -Force`, повторить.
5. Fallback: `seo-writer/scripts/md2docx.py`

### Шаг 5 — Финал

- Обновить `README.md`: диапазон номеров, таблица новой восьмёрки.
- Выдать сводку: NN, заголовок, объём лонгрида, пути к трём форматам.
- Проверить счётчики: `ТЗ` = `Лонгриды Дзен` = `ТГ Посты` = `DOCX`.

## Оригинальность (без антиплагиата)

Антиплагиат **не запускаем**. Уникальность обеспечивается **на этапе написания**.

### Референсы — только идеи

Lily Ray, Gaetano, Aleyda, треды X.com, vc.ru — источник **угла и тезиса**, не текста.
Запрещено: копировать формулировки, абзацы, структуру чужой статьи 1:1, переводить
англоязычный пост дословно.

### Что делает текст «своим»

В каждом материале — минимум **два** оригинальных элемента из списка:
- авторский чеклист или набор промптов (свои номера, свои формулировки);
- таблица / схема с уникальными колонками под тему ТЗ;
- сценарий B2B или ниша Artkod (машиностроение, стоматология, YMYL);
- российский контекст (Алиса, 2ГИС, Дзен, vc.ru — не как декорация, а в механике);
- собственная метафора или рамка («gap-диагноз», «dark funnel» — переосмыслить, не цитировать).

### Лонгрид vs Telegram

- **Лонгрид** — развёрнутая экспертиза: примеры, шаги, таблицы. Не сжатый пересказ референса.
- **ТГ-пост** — **другой угол**, не вступление к лонгриду. Свой хук, одна мысль, другой заход.

### Внутри восьмёрки

- Не повторять одни и те же примеры, метафоры и списки между NN 41–48.
- Перед финалом субагент сверяет драфт с `references/originality.md`.

### Редакторы

`plan-editor` и `article-editor` (лонгрид), `post-editor` (ТГ) — отклоняют шаблонный
«индустриальный» текст без конкретики Artkod и без оригинальных блоков из ТЗ.
