# Оригинальность при написании (без антиплагиата)

Субагент читает этот файл перед финальной сборкой лонгрида и ТГ-поста.

## Принцип

Референсы дают **мысль**, не **текст**. Пиши так, будто объясняешь коллеге из Artkod
свой взгляд — не пересказываешь чужую статью или тред.

## Обязательно в каждом NN

1. **Своя структура примеров** — не копировать порядок H2 из референса; следовать ТЗ.md.
2. **Минимум 2 оригинальных блока** — чеклист, таблица, набор промптов, сценарий,
   мини-кейс по названию (без цифр результата), сравнение платформ РФ.
3. **Свои формулировки** — переформулировать общие определения GEO/AEO; не брать
   готовые абзацы с vc.ru, Habr, англоязычных блогов.
4. **ТГ ≠ лонгрид** — пост с другим хуком и одной отдельной мыслью; не первый абзац статьи.

## Запрещено

- Дословный или близкий перевод англоязычных постов.
- Копипаст фраз из референса в заголовки, хуки, списки.
- Один и тот же шаблонный вступительный абзац («В 2024 году нейросети изменили SEO…»).
- Повтор одних и тех же 3–5 промптов, что уже есть в других NN проекта (сверять с `ТЗ/`).
- Generic listicle без привязки к B2B / Artkod / РФ.

## Самопроверка перед сдачей

- [ ] Могу ли я назвать 2 блока, которых нет в референсе из ТЗ?
- [ ] Хук ТГ-поста отличается от H1 и первого абзаца лонгрида?
- [ ] Есть ли конкретика: промпт, шаг, таблица, ниша — не только определения?
- [ ] Нет ли дежурных AI-вступлений и пересказа Princeton GEO без своего вывода?

Если два «нет» — переписать слабые блоки, не сдавать.
