# Умение: Анализатор чатов и постановщик задач (chat-analyzer)

## Описание
Используй этот навык, когда Алекс присылает текст переписки, расшифровку аудио, лог созвона или пересланные сообщения и просит «разбери», «вытащи задачи», «сделай выжимку».

## Инструкция по выполнению (СТРОГИЙ АЛГОРИТМ):

Когда срабатывает этот навык, ты обязан выдать ответ, состоящий строго из 4 блоков:

1. 📋 **Саммари**: Краткая выжимка (в чем суть обсуждения, 2-3 предложения).
2. 📝 **Мои задачи (Алекс)**: Что лично Алексу нужно сделать (отписать в комментарии, проконтролировать лично, занести в Google Календарь).
3. 👥 **Задачи для команды**: Черновики задач для постановки в Bitrix24. Для каждой задачи чётко укажи:
   - Название
   - Описание
   - Ответственного сотрудника (сверься с `MEMORY.md` для получения правильных имен)
   - Дедлайн
   - Проект (группу в B24)
4. ❓ **Ожидание апрува**: Явный вопрос: "Алекс, жду команду 'Ок' или 'Создавай', чтобы закинуть задачи в Битрикс."

## ⛔ КРИТИЧЕСКОЕ ПРАВИЛО (Draft First)
НИКОГДА не вызывай API Битрикс24 (добавление или изменение задач) на этом этапе. Сначала покажи черновики. Только после того, как Алекс в следующем сообщении напишет «Да», «Ок» или «Делай», ты можешь физически создавать задачи.

## Интеграция с Obsidian
Если в переписке есть новые доступы, кейсы, КП или регламенты, действуй согласно правилам из `/root/.openclaw/workspace/obsidian_vault/wiki/Мои Проекты/Арткод/chat_analysis_protocol.md` и предложи сохранить их в нужные папки.
