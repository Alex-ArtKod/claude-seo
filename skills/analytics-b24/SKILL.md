# Analytics-B24

Скилл для анализа эффективности команды по проекту из Битрикс24 за произвольный период (по запросу, не по крону), сопоставления результатов со стратегией и планирования будущих шагов.

## ⚙️ Рабочий процесс (Workflow)

Запускается по запросу вида:
*«Сделай статистику по проекту Контраст Дизайн за период с 01.06 по 30.06.2026»*

1. **Резолв клиента:** имя проекта из запроса резолвится через канонический реестр `wiki/Мои Проекты/Арткод/b24_client_registry.md` (сгенерирован из `scripts/b24_pipeline/config/clients.json`) — это защита от повтора исторического бага с тремя дублирующими папками «Вира». Если период не назван явно — уточнить у Алекса, дефолт не подставляется автоматически.
2. **Сбор и обогащение:** запускается канонический пайплайн:
   ```bash
   node scripts/b24_pipeline/index.js "<CanonicalProjectName>" <DATE_FROM> <DATE_TO>
   ```
   Скрипт тянет все задачи группы (`tasks.task.list`, фильтр только по `GROUP_ID`), затем батчем честно дотягивает комментарии, результаты и записи времени (`task.commentitem.getlist`, `tasks.task.result.list`, `task.elapseditem.getlist`) — без моков и кэш-фоллбэков.
3. **Чтение стратегии:** читается `strategy.md` в папке клиента (для большинства — `wiki/Клиенты/{Проект}/strategy.md`, для внутреннего проекта «Арткод» — `wiki/Мои Проекты/Арткод/strategy.md`, путь берётся из `wiki_path` в реестре). Формат — единый шаблон v2 (frontmatter + теги `[G#]`/`[K#]`/`[P-...]`), см. `docs/plans/breezy-dreaming-liskov.md`.
4. **Сведение со стратегией:** пайплайн сопоставляет задачи периода с пунктами плана (точно — по явным `<!-- b24-ref: ID -->`, иначе — эвристикой по ключевым словам, с пометкой «предположительно»). Финальную интерпретацию сопоставления даёт ассистент в чате поверх структурированной подсказки скрипта.
5. **Запись:** результаты сохраняются в `wiki/Клиенты/{Проект}/otchety/{YYYY}/otchet_{DD.MM.YYYY}_{DD.MM.YYYY}.md` (не перезаписывается), живой снепшот всех задач — в `tasks.md` (перезаписывается каждый раз), сырой JSON-снепшот периода — в `raw/Клиенты/{Проект}/{YYYY}/` (не коммитится, см. `.gitignore`).
6. **Резюме в чат:** ассистент выводит краткие выводы и кликабельную ссылку на сохранённый отчёт.

## 💡 Управление сессиями и бюджетом

Для экономии ИИ-бюджета и поддержания легкого контекста основного чата (< 100k токенов) используй следующие команды:
*   `/new` — архивирует текущий диалог и начинает новую чистую сессию.
*   `/reset` — полностью очищает историю текущей сессии на месте.

Это позволяет сбросить накопленный контекст перед проведением дальнейших доработок или тестирования.

## 🚫 Правила работы с задачами (обязательно)

* **Не удаляем, не редактируем и не ставим задачи без аппрува или прямой команды Алекса.** Аналитика и отчёты — это read-only операции по умолчанию; любое изменение состояния задач в Битрикс24 требует явного одобрения.
* **«Задачи за период»** — это задачи, которые за выбранный период:
  * хотя бы раз были запущены (был активный таймер/трек времени) в этом периоде, **или**
  * дедлайн по ним попадает в этот период.

  Выборка должна учитывать оба условия (объединение, не пересечение).
* **Обязательные поля при постановке любой задачи:** ответственный, дедлайн, описание. Задача без одного из этих полей не создаётся — сначала уточняем недостающее у Алекса.

## 🛡️ Безопасность и отказоустойчивость

* **Без fallback на моки.** Если Bitrix24 REST недоступен (истёкшая подписка, сетевая ошибка, `insufficient_scope`) — пайплайн останавливается с явной ошибкой в консоли. Никакого локального кэша с тестовыми данными: старая версия скрипта имела такой fallback с захардкоженными фейковыми задачами, и это реально привело к тому, что мок-данные один раз попали в вики как будто настоящие факты (см. баннер в `wiki/Клиенты/Вира-АртСтрой/otchety/2026/otchet_legacy_03.06.2026.md`). Если API недоступен — сообщи Алексу об этом напрямую, не подставляй заглушки.
* **Резолвинг ФИО:** вебхук студии не имеет права на `user.get` (`insufficient_scope`) — имена ответственных резолвятся через статическую карту `scripts/b24_pipeline/config/team.json`. Если в отчёте встречается «Неизвестный сотрудник (ID N)» — этого человека нет ни в live-ответе Битрикса, ни в `team.json`; уточни у Алекса и дополни файл.
