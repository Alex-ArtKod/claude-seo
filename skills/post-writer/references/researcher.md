# Researcher — ресёрч и web fetch для поста

Цель: заземлить пост на свежую конкретику (цифры, источники, обновления платформ).
Поле AI-поиска динамичное — данные устаревают за недели, поэтому проверяем свежесть.

## Когда искать в вебе
- В посте есть фактический claim, которого нет в `rules.md`: цифра, дата, название
  исследования, обновление платформы (ChatGPT/Perplexity/Gemini/Google AIO/Алиса),
  чужой кейс, статистика рынка.
- Тема привязана к «что нового» — тогда ищем апдейты за последние 1–3 месяца.
- Автор дал ссылку-источник на входе — фетчим её и достаём факты/цифры/цитаты.

## Когда НЕ искать
- Факт уже есть в `rules.md` (он первичен).
- Утверждение общеизвестное и не числовое («E-E-A-T важен для YMYL»).
- Пост чисто мненческий/продающий без проверяемых claim.

## Как искать
1. `WebSearch` по теме + маркер свежести (год/месяц, «2026», «update», «исследование»).
2. Для конкретного источника или ссылки автора — `WebFetch` с точным вопросом
   («какие цифры по приросту кликов», «дата обновления», «что именно изменилось»).
3. Брать **первоисточник** (блог платформы, сайт исследования), а не пересказ.
4. Фиксировать на каждый факт: значение + источник (URL/название) + дата.

## Что вернуть в пайплайн
Список фактов в формате:
```
- {факт/цифра} — источник: {название + URL} — дата: {когда опубликовано}
- свежесть: {актуально / устарело / спорно}
```
Плюс 1–2 «цепляющих» конкретных детали (число, кейс, цитата апдейта), которые
можно вшить в пост для живости.

## Заземление на цифру
Голос автора всегда опирается на число или названный источник (см. voice-profile
п.10). Если ресёрч не дал ни одной конкретной цифры/источника — отметь это:
пост рискует получиться «водой», сигналь в insight-gate.
