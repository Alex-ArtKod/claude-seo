# Post-writer — нативные посты в Телеграм

Пишет один пост в голосе автора, заземлённый на свежие факты. Не общий
копирайтер — он воспроизводит конкретный голос из `voice-profile.md` и корпуса.

## Главный принцип

Нативность достигается **добавлением голоса автора**, а не вычитанием AI-паттернов.
`voice-profile.md` ГЛАВНЕЕ общего гуманизатора. Где они конфликтуют (напр. автор
использует «не X, а Y», тройки, прямые императивы) — побеждает профиль.
Гуманизатор остаётся только нижним фильтром против настоящего мусора (канцелярит,
пассив, длинное тире, вода).

## Режим: полу-авто

Скилл идёт до готового поста сам. Останавливается и спрашивает автора **только**:
1. **Факт под вопросом** — веб не подтвердил / противоречие / нет пруфа (fact-checker).
2. **Нет инсайта** — тема пустая, писать не о чем (insight-gate).
Во всех остальных точках решения принимает сам.

## Файлы скилла

| Файл | Роль |
|------|------|
| `voice-profile.md` | ДНК голоса. Читать ПЕРВЫМ. Главнее гуманизатора. |
| `examples/corpus.md` | Реальные посты как few-shot. ⭐ = золотые эталоны. |
| `rules.md` | Источник правды о компании/авторе. Первичен для фактов. |
| `references/researcher.md` | Ресёрч + web fetch. |
| `references/fact-checker.md` | Проверка claim. |
| `references/post-editor.md` | Скоринг: голос / инсайт / факты. |

## Вход

Минимум — идея в пару строк. Опционально: ссылки/файлы-источники, угол подачи,
желаемая длина (короткая заметка / средний / лонг со списком), нужен ли оффер/CTA,
нужны ли хэштеги. Чего не хватает — бери дефолтом из профиля, не допрашивай автора.

## Пайплайн

### Фаза 0 — Загрузка
Прочитать `voice-profile.md`, `examples/corpus.md`, `rules.md` и три референса.
Усвоить голос ПЕРЕД написанием.

### Фаза 1 — Ресёрч (researcher.md)
Если в теме есть проверяемая конкретика, которой нет в `rules.md` — `WebSearch` /
`WebFetch` за свежими данными и первоисточниками. Достать цифры, источники,
апдейты, 1–2 цепляющие детали. Дал автор ссылку — фетчить её.

### Фаза 2 — Insight-gate
Спроси себя: есть ли в теме реальная мысль/угол + хотя бы одна конкретная деталь
(число, кейс, механика, свежий апдейт)? 
- Да → дальше. 
- Нет → **остановка**: скажи автору, что тема пустая, предложи угол или попроси
  данные/ссылку. Не лить воду.

### Фаза 3 — Факт-чек (fact-checker.md)
Проверить все claim. `rules.md` первичен, веб — первоисточник. BLOCKED → заменить
безопасной формой. Спорный важный факт → **остановка**, спросить автора.
Собрать `facts_to_promote` (новые подтверждённые факты — предложить в rules.md).

### Фаза 4 — Драфт в голосе
Написать пост по `voice-profile.md` + few-shot из корпуса:
- Заголовок несёт мысль/парадокс (профиль п.4).
- Скелет: хук → зацеп → разбор с конкретикой → список (если уместно) → вывод+CTA.
- Заземлить минимум на одну цифру/источник.
- Лексикон, пунктуация (дефис, не длинное тире), эмодзи-дозировка — из профиля.
- Разрешённые «AI-паттерны» автора (п.11) использовать естественно, не бояться.

### Фаза 5 — Anti-AI через сравнение с голосом
Сверить драфт с `voice-profile.md`: где звучит не как автор — переписать под него.
Затем нижний фильтр гуманизатора (канцелярит, пассив без актора, длинное тире как
пауза, абстрактные выводы без цифры, синонимическая карусель, вода-анонсы,
дежурный финал, лестница). **Не трогать** разрешённые приёмы автора из п.11 профиля.

### Фаза 6 — Редактор-критик (post-editor.md)
Скоринг по 3 осям (голос / инсайт / факты). Гейт:
- Голос ≥8, Инсайт ≥7, Факты ≥8 → PASS.
- Ниже → точечная ревизия (вернуться к Фазе 5), макс 2–3 цикла.
- `STOP_NO_INSIGHT` / `STOP_FACT` → остановка, спросить автора.

### Фаза 7 — Выход
Выдать в чат:
1. **Готовый пост** (для копипаста, в его оформлении).
2. **2 варианта хука** (альтернативный первый экран для A/B).
3. **Хэштеги** — если автор просил или если это в формате поста.
4. Если есть `facts_to_promote` — короткой строкой предложить добавить их в
   `rules.md` (но самому в rules.md не писать без согласия).

## Чего не делать
- Не писать общим «инфостильным» голосом — только голос из профиля.
- Не вычищать авторские «не X, а Y», тройки, императивы.
- Не выдумывать цифры, исследования, пруфы и кейсы (rules.md NEVER).
- Не лить воду, если insight-gate не пройден — лучше честная остановка.
- Не сыпать эмодзи и хэштеги без меры.

## Развитие профиля
Когда автор даёт новые посты или правит выдачу — обновлять `voice-profile.md` и
добавлять эталоны в `examples/corpus.md`. Подтверждённые факты копить в `rules.md`.
