# Парсер чатов Telegram (Рабочий процесс сегментированного анализа)

Этот навык предоставляет рабочий процесс для парсинга крупных HTML-экспортов чатов Telegram в удобные для обработки сегменты простого текста (до 4000 строк каждый) и выполнения высококачественного анализа по этим сегментам.

## Процесс работы

1. **Конвертация HTML в TXT:** Конвертирует все файлы `messages*.html` в целевой папке с сырыми данными чата в несколько плоских файлов `parsed_chat_N.txt` (макс. 4000 строк в файле).
2. **Сегментированный анализ:**
   - Анализ выполняется ровно по **одному сегменту** (до 4000 строк) за сессию агента.
   - В папке чата ведется `log.md` для отслеживания прогресса.
   - При каждом запуске агент считывает `log.md`, чтобы определить следующий сегмент для обработки.
3. **Отчетность:** Кумулятивные результаты обновляются в `wiki/chats/<имя_чата>/chatsInfo.md`.

## Быстрый старт

### Шаг 1: Конвертация HTML в TXT

Передайте путь к папке с сырыми данными чата, содержащей ваш экспорт Telegram:

```bash
bash /root/.openclaw/workspace/skills/tg-chat-parser/scripts/run.sh <путь_к_папке_с_сырым_чатом>
```

**Пример:**
```bash
bash /root/.openclaw/workspace/skills/tg-chat-parser/scripts/run.sh /root/.openclaw/workspace/obsidian_vault/raw/chats/chat_002
```

### Шаг 2: Анализ следующего сегмента

Дайте указание агенту OpenClaw проанализировать чат. Агент автоматически:
1. Проверит наличие `log.md` в папке чата.
2. Определит следующий файл `parsed_chat_N.txt`.
3. Выполнит анализ (инсайты SEO/GEO, контакты, задачи).
4. Обновит `log.md` новым статусом (например, "Обработан файл 2, остановка на строке 8000").
5. Обновит `wiki/chats/<имя_чата>/chatsInfo.md` найденными данными.

## Отслеживание прогресса (log.md)

Файл `log.md` в папке с сырыми данными чата должен иметь следующий формат:
```markdown
# Журнал анализа
- **Последний файл:** parsed_chat_1.txt
- **Всего обработано строк:** 4000
- **Статус:** В процессе / Завершено
```

## Инструкции для агента при выполнении анализа:

- **Чтение журнала:** Начинайте с чтения `<target_dir>/log.md`. Если он отсутствует, начинайте с `parsed_chat_1.txt`.
- **Ограничение:** Не обрабатывайте более одного файла `parsed_chat_N.txt` (приблизительно 4000 строк) за один сеанс/ход.
- **Обновление журнала:** Обновляйте `log.md` немедленно после анализа. Сообщайте о прогрессе (обработанные файлы, количество строк) как в самом файле, так и пользователю.
- **Дополнение результатов:** Добавляйте новые инсайты в `wiki/chats/<имя_чата>/chatsInfo.md`, не перезаписывая предыдущие важные данные (если только не выполняется исправление или дедупликация).

## Ссылки на скрипты

* `scripts/run.sh`: Основная точка входа для фазы конвертации и разделения.
* `scripts/html_to_txt.js`: Считывает все файлы `messages*.html` и создает `parsed_chat_1.txt`, `parsed_chat_2.txt` и т.д.
