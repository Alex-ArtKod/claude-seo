# Project Intake (Artkod)

Заводит новый клиентский проект: интервью → папка проекта → пошаговая стратегия
под конкретный проект. Костяк (то, что не меняется от проекта) берётся из папки `references/` внутри
скилла (копия «Системы»); под проект дорабатываются только 🟨-части (ниша, регион,
площадки, промпты).

Главный принцип: **80% предопределено костяком, 20% уточняется в интервью.**
Не переписывай стратегию с нуля — специализируй костяк.

---

## Пути (скилл самодостаточный / портативный)

- **Костяк — внутри скилла:** папка `references/` рядом со `SKILL.md`. Переносится вместе со
  скиллом, менять ничего не нужно. Файлы: `references/00_Классификатор проекта.md`,
  `references/01_Матрица маршрутизации.md`, `references/02_Шаблон стратегии SEO-GEO-AEO.md`,
  `references/03_Распределение работ и роли.md`, `references/04_Шаблон карты сущностей.md`,
  `references/05_Вводные и доступы.md`.
- **`CLIENTS_DIR`** (куда создавать папки клиентов) = `C:\Users\Алекс\Desktop\claude\raw\Клиенты`.
  Это единственный путь, привязанный к компьютеру. Если такой папки нет (например, скилл
  перенесён на другой комп) — **спроси у пользователя, куда складывать проекты**, используй
  его ответ и дальше применяй его же.

> Костяк в `references/` — это копия папки «Система». Если правишь «Систему» в Obsidian,
> раз в какое-то время обнови копию: скопируй 6 .md-файлов в `references/`.

## Invocation

```
/project-intake [Клиент] [услуга]
```

- `[Клиент]` — название клиента/проекта (если нет — спросить одной строкой).
- `[услуга]` — одно из: `SEO`, `SEO+GEO+AEO`, `GEO+AEO` (если нет — спросить, это первый вопрос).

Если аргументов нет — начни с интервью (Шаг 1), не падай.

---

## Шаг 0. Подготовка

1. Прочитай костяк из папки `references/` рядом со `SKILL.md` (минимум файлы 02, 03, 04, 05 — они шаблоны выхода).
2. Определи `Клиент` и `Услугу`. Услуга задаёт ветку стратегии:
   - `SEO` → SEO-трек.
   - `GEO+AEO` → GEO-трек (карта сущностей и внешний контур в центре).
   - `SEO+GEO+AEO` → комбинированный трек.

---

## Шаг 1. Интервью

Спрашивай **группами** (не по одному вопросу). Используй `AskUserQuestion` для выборов
и свободный текст для остального. Не выпытывай то, что можно поставить по умолчанию —
предложи дефолт и иди дальше. Дефолты явно называй.

**Группа A. Разметка (классификатор, файл 00):**
- Тип бизнеса: B2B-PROD / LOCAL / ECOM / SERVICE
- География: GEO-1 / GEO-2-3 / GEO-RF / GEO-LOCAL
- Состояние сайта: SITE-NONE / SITE-OK / SITE-BAD / SITE-OTHER (+ URL сайта)
- Цель/KPI: главная + сопутствующая (подскажи типовые под услугу: SEO→KPI-POS/TRAF, GEO→KPI-AI)
- Тариф: T-SEO-BASE / T-GEO-1 / T-GEO-2 / T-GEO-3 / T-MIX / T-DIRECT
- Особенности: по умолчанию **+5ч тех.поддержки/мес**; спроси про ограничения движка, запреты на данные, договорённости продаж.

**Группа B. Вводные и доступы (файл 05):**
- Какие доступы уже есть (остальные из обязательного списка пойдут в «запросить»).
- Есть ли: прайс, приоритетные направления, конкуренты, эксперты, кейсы/награды, разрешение на медиа-контент, фактура для отзывов.

**Группа C. Ниша-специфика (для 🟨-блоков):**
- Для SEO: направления семантического ядра, приоритетные коммерческие страницы, регионы под страницы.
- Для GEO/AEO: ключевые направления под промпты; отраслевые площадки (предложи набор под нишу, уточни); эксперты/спикеры; иностранные площадки нужны (для тревел/международных — да).
- Для комбинированного — обе подгруппы.

Держи интервью коротким: 2–3 захода `AskUserQuestion` + пара уточнений текстом. Собери ответы.

## Шаг 4. Специализация стратегии (главная логика)

Возьми костяк из файла 02 и собери **пошаговый план на 8 недель под выбранную услугу**.
Не копируй весь костяк — сделай конкретный план с ответственными и скиллами. Для каждого
шага укажи: **что делаем · ответственный (из файла 03) · скилл Claude (если есть) · точка касания клиента**.

### Ветка SEO (SEO-логика)
Порядок: сбор ядра → исправление всех ошибок → постраничная оптимизация → SEO-контент под ядро → внешнее/ссылки → отчёт.
- Н1: сбор ядра и контекста — `seo-context`, `seo-cluster`; срез позиций/трафика; доступы.
- Н2: тех.аудит — `seo-audit`; карта структуры сайта.
- Н3: исправление ошибок (программисты) + ТЗ постранично — `seo-tz-generator`.
- Н4: постраничная оптимизация + первые тексты — `seo-writer`; отчёт за месяц.
- Н5–6: SEO-контент под ядро, перелинковка, E-E-A-T.
- Н7: внешние публикации/ссылки; ТЗ.
- Н8: контроль индексации/позиций + план на 2 мес.

### Ветка GEO+AEO (GEO-логика, эталон Вира-АртСтрой)
Порядок: промпты → срез → карта сущностей → внешние (+иностранные) → Wikidata → репутация.
Объём: **10–15 текстов/мес** под захват промптов (комбинируем только очевидно комбинируемые), ~10 материалов/мес.
- Н1: подбор промптов; срез AI-видимости — `seo-geo` (вкл. GigaChat); регистрация в Bing Вебмастер; стратегия на 2 мес.
- Н2: карта сущностей (файл 03 проекта) + сверка NAP; первые ТЗ на внешние площадки — `/geo-tz-generator`; импорт в Bing Места из Google Business.
- Н3: стратегия отзывов на карты (согласование); статьи в блог; внешние рейтинги.
- Н4: тех/SEO-аудит под GEO/AEO; коммерческие страницы (E-E-A-T, 152-ФЗ формы); публикация отзывов; отчёт.
- Н5: ТЗ на устранение ошибок; заявка на Wikidata-сущность; 2 статьи в блог.
- Н6: E-E-A-T анализ; блоки (Команда/Авторы/Награды/Кейсы/Гарантии/Документы/FAQ); страницы авторов.
- Н7: рекомендации по блогу; ТЗ на внешние статьи/рейтинги; работа с отзывами; граф знаний на «О компании».
- Н8: публикация внешних статей; буст бренда в рейтингах на первые места; экспертные материалы в блог.
Иностранные площадки (Substack/Medium/Quora, англ.) — обязательный элемент; для travel обязательно.

### Ветка SEO+GEO+AEO (комбинированный)
- Если `SITE-NONE`/`SITE-BAD` — сначала DEV-этап (программисты + дизайн), SEO/GEO стартуют после приёмки.
- Далее оба трека параллельно: SEO держит структуру/ядро/постраничную оптимизацию, GEO — промпты/карту сущностей/внешний контур. PM (Симакова) дирижирует, Козлитин — владелец продукта.
- Сохрани ритм касаний каждые 3–4 дня и запрос фактуры заранее.

В конце файла 02 добавь раздел **«Ответственные и ритм»** (кратко из файла 03) и
**«Что запросить у клиента»** (из файла 01 проекта).

## Шаг 5. Итоговое резюме (в чат)

Коротко выведи:
1. Путь к созданной папке проекта.
2. Разметку (услуга + 6 осей) одной строкой.
3. Список **обязательных доступов и вводных к запросу** у клиента.
4. Первые 3 действия и какие скиллы запускать (`seo-context` / `/geo-tz-generator` / `seo-writer`…).
5. Напоминание про карту сущностей (заполняется вручную) и ритм касаний 3–4 дня.

## Правила

- Факты/цифры/юр.данные не выдумывай — если нет в ответах клиента, ставь плейсхолдер `‹уточнить›` и вынеси в «запросить».
- Не дублируй весь костяк в файлы проекта — специализируй и ссылайся на «Систему».
- Если `[услуга]` не из трёх вариантов — уточни (пока поддерживаем только эти три).
- Всё, что генерируешь в папку проекта — на русском.
- `/geo-tz-generator` и часть GEO-пайплайна — внутренние скиллы Artkod (могут быть ещё не подгружены с сервера); в стратегии указывай их как штатные звенья.
