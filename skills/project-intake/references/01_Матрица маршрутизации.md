# 01. Матрица маршрутизации проектов Арткод

> Назначение: превращает разметку из [[00_Классификатор проекта]] в готовый сценарий работы. Нашёл строку под свой тип проекта — получил ответственного, пайплайн, этапы, KPI и шаблон КП. Это и есть «80% понимания на входе».
>
> Читается так: **ось A (услуга)** задаёт основной сценарий, оси B–F уточняют детали (кейсы, семантику, дедлайны, тариф).

## Основная матрица: услуга → сценарий

### A = SEO (классическое SEO)
- **Ведёт:** Козлитин (стратегия) + SEO-спец (исполнение), PM Симакова (клиент)
- **Пайплайн (SEO-логика):** сбор ядра → тех.аудит и исправление всех ошибок → постраничная оптимизация → SEO-контент под ядро → внешнее/ссылки → отчёт
- **Claude-звенья:** `seo-audit` → `seo-context` → `seo-cluster` → `seo-tz-generator` → `seo-writer`
- **Типовой KPI:** `KPI-POS` / `KPI-TRAF` (для B2B-PROD часто позиции в ТОП-10 по графику)
- **Тариф:** `T-SEO-BASE` (от 52к); при 2–3 регионах — региональная тарификация как `T-MIX` (индивидуально)
- **КП/договор:** шаблон SEO (Google Docs, «Шаблон договора на SEO (Чистый)»); ориентир — договор Элимет

### A = GEO / AEO
- **Ведёт:** Козлитин + SEO-спец, PM Симакова
- **Пайплайн (GEO-логика):** срез AI-видимости → сбор промптов (~100) → ТЗ через `/geo-tz-generator` → **10–15 текстов/мес** под захват промптов → публикация на сайте + внешних площадках (вкл. иностранные бесплатные: Substack/Medium, англ.) → замер цитируемости в ИИ
- **Правило комбинирования:** комбинируем только очевидно комбинируемые промпты; под остальные — отдельный текст под каждый.
- **Claude-звенья:** `seo-geo` (срез) → `seo-context` → `/geo-tz-generator` → `seo-writer`
- **Типовой KPI:** `KPI-AI` (упоминания/цитирование в нейросетях) + `KPI-TRAF`
- **Тариф:** `T-GEO-1/2/3` (64 / 112 / 262к)
- **КП:** GEO/AEO (ЛаВояж, Мебель-Москва); кейсы — «Круизный Лайнер» (ТОП-1 → ИИ-выдача), «Мало Места». Для ЛаВояж иностранные площадки — обязательно.

> ⚠️ `/geo-tz-generator` и часть GEO-пайплайна — внутренние скиллы Artkod, пока не подгружены с сервера (Git). В системе фигурируют как штатные звенья.

### A = MAPS (Яндекс.Карты / 2ГИС / Zoon)
- **Ведёт:** SEO-спец + ассистент (наполнение), PM Симакова
- **Пайплайн:** аудит карточек → NAP-консистентность → наполнение/оптимизация → отзывы (SERM) → гео-грид замер
- **Claude-звенья:** `seo-maps` / `seo-local`; отзывы — `seo-rewies-writer`
- **Типовой KPI:** `KPI-LEAD` / `KPI-REP`; условие — `GEO-LOCAL`
- **Тариф:** обычно часть `T-MIX` или пакета GEO

### A = DEV (разработка)
- **Ведёт:** Сысоев / Даблшевич + Артём (дизайн), PM Симакова
- **Когда обязателен:** `SITE-NONE` или `SITE-BAD` — идёт первым этапом перед SEO
- **Пайплайн:** ТЗ на разработку → дизайн → вёрстка/сборка WP → приёмка → передача в SEO
- **Тариф:** почасовая (ориентир 3000 руб./час разработка, 2300 руб./час дизайн — как у Стальтека)
- **Примечание:** непрофильно/неохотно — по возможности брать как довесок к SEO, а не самостоятельно

### A = PPC (Директ)
- **Ведёт:** Вика / Яна
- **Пайплайн:** сбор семантики → кампании → маркировка рекламы (напрямую или через eLama) → ведение → отчёт
- **Типовой KPI:** `KPI-LEAD`
- **Примечание:** не фокус; брать как поддержку SEO/GEO

### A = SERM (репутация)
- **Ведёт:** SEO-спец / ассистент + PM
- **Пайплайн:** аудит репутации → стратегия площадок → генерация/размещение отзывов → мониторинг
- **Claude-звенья:** `seo-rewies-writer`
- **Типовой KPI:** `KPI-REP`
- **Тариф:** `T-MIX` или отдельно

### A = MIX (комплекс)
- Собирается из строк выше. Правило: **DEV → SEO/GEO → MAPS/SERM → PPC** (сначала фундамент, потом продвижение, потом поддержка).
- **Ведёт:** Козлитин как владелец продукта + PM Симакова как дирижёр этапов.
- **Тариф:** `T-MIX`, ориентиры — региональные пакеты Стальтека / комплекс Мебель-Москва.

## Как оси B–F меняют сценарий (модификаторы)

| Ось | Значение | Что меняет |
|-----|----------|-----------|
| B | `B2B-PROD` | KPI обычно позиции; семантика — номенклатура/ГОСТы; кейсы Элимет/Стальтек |
| B | `SERVICE` (тревел) | упор в GEO/AEO и контент; кейс ЛаВояж/Круизный Лайнер |
| B | `LOCAL` | обязательно MAPS + SERM; кейс Мало Места |
| C | `GEO-2-3` / `GEO-RF` | региональная структура сайта → тариф считается как `T-MIX` (индивидуально), +нагрузка на DEV |
| D | `SITE-NONE` / `SITE-BAD` | DEV идёт первым этапом, SEO стартует после приёмки |
| E | `KPI-AI` | обязательно GEO-пайплайн и замер цитируемости в ИИ |
| F | `T-GEO-3` (262к) | максимальный объём: SEO+GEO+MAPS, выделенный SEO-спец |

---

## Быстрые примеры (по реальным клиентам)

- **Элимет** = SEO · B2B-PROD · GEO-1 · SITE-OK · KPI-POS · T-SEO-BASE → сценарий «A=SEO», Козлитин+спец, договор Элимет как шаблон.
- **ЛаВояж** = GEO · SERVICE · GEO-RF · SITE-OK · KPI-AI · T-GEO-2 → сценарий «A=GEO», кейс Круизный Лайнер в КП.
- **Стальтек** = MIX (SEO+DEV+дизайн) · B2B-PROD · GEO-2-3 · SITE-BAD · KPI-POS · T-MIX → сценарий «A=MIX», порядок DEV→SEO, региональная тарификация (индивидуально).
- **Мебель-Москва** = GEO+MAPS · LOCAL · GEO-LOCAL · KPI-LEAD · T-GEO-2 → GEO + карты + SERM, кейс Мало Места.
